## Requirements

### Purpose

Publish a consumable Rust and Go SDK patch release for the completed
SOW-0127 through SOW-0131 work, so downstream Netdata work can move existing
SDK consumers to `0.7.7` without guessing about local commits.

Fit-for-purpose target:

- Rust crates and Go module tags point to the same release commit.
- Public install snippets and Rust package metadata name `0.7.7`.
- The release includes completed container-aware host identity, engine cache
  namespace/version discipline, stateful unique enumeration, sync-on-archive
  opt-out, and directory unique cache parity work.
- This SOW does not perform broader Netdata integration. Existing Netdata SDK
  consumers can be bumped to `0.7.7` by the separate Netdata worker.
- Containerized Netdata callers that intentionally use host identity through
  the Go `journalhost` helper need to pass the host filesystem prefix, such as
  `/host`, so machine-id resolution prefers host machine-id files.

### User Request

The user requested releasing `0.7.7`, confirmed that broader Netdata
integration should not be done in this repository now, and asked whether
Netdata plugins running in containers need to pass a host prefix for correct
machine-id behavior.

## Implications And Decisions

1. Release version
   - Decision: release `0.7.7`.
   - Evidence: `v0.7.6` / `go/v0.7.6` already exist and the completed
     SOW-0127 through SOW-0131 commits are after them.
   - Classification: surgical.
   - Implications: Rust metadata, docs snippets, crates.io publication, root
     tag, and Go submodule tag all target `0.7.7`.
   - Risks: if `0.7.7` is published with wrong metadata or wrong tag target,
     the version cannot be repaired by retagging without explicit destructive
     approval and possible proxy/cache fallout.

2. Netdata integration scope
   - Decision: do not do broader Netdata integration in this repository as part
     of this SOW.
   - Evidence: the user stated another worker owns that work and only current
     Netdata SDK users need to move to `0.7.7`.
   - Classification: surgical.
   - Implications: this SOW publishes SDK artifacts only; Netdata source changes
     are outside this repository and outside this SOW.
   - Risks: Netdata will not consume `0.7.7` until the separate worker updates
     its dependency pins.

3. Container host identity
   - Decision: Netdata callers using `journalhost` from a container should pass
     `HostFilesystemPrefix`, for example `/host`, when they intentionally want
     the host machine ID.
   - Evidence: Go helper implementation and docs show host-prefixed machine-id
     files are checked before container-local paths only when the option is set.
   - Classification: long-term-best.
   - Implications: containerized writers avoid silently writing under the
     container machine-id namespace when host identity is intended.
   - Risks: if `/host/etc/machine-id` exists but is invalid, the helper returns
     an error by design instead of falling back silently.
