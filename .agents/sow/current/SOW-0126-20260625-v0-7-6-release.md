## Implications And Decisions

1. Release version
   - Decision: release `0.7.6`.
   - Evidence: `v0.7.5` / `go/v0.7.5` already exist remotely and peel to
     `d338e13`, while the SOW-0124 fix is at `203f248`.
   - Classification: surgical.
   - Implications: Rust metadata, docs snippets, crates.io publication, root
     tag, and Go submodule tag all target `0.7.6`.
   - Risks: if `0.7.6` is published with wrong metadata or wrong tag target,
     the version cannot be repaired by retagging without explicit destructive
     approval and possible proxy/cache fallout.

2. Release gate
   - Decision: run release validation and read-only reviewer checks before
     irreversible publication.
   - Evidence: prior release SOWs used local validation plus external review
     before pushing public artifacts.
   - Classification: long-term-best.
   - Implications: publication happens only after tests, dry-runs, and reviewer
     gate are clean.
   - Risks: release takes longer, but avoids publishing broken permanent
     artifacts.
