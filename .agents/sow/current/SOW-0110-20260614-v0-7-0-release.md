# SOW-0110 - v0.7.0 Release

## Status

Status: in-progress

Sub-state: release preparation started on 2026-06-14.

## Requirements

### Purpose

Publish a consumable v0.7.0 release for Rust and Go consumers after the
reader-window verifier repair and Python/Node parity reconciliation, so
Netdata integration can depend on a tagged SDK instead of a moving branch.

### User Request

Release `0.7.0` for Netdata integration.

### Assistant Understanding

Facts:

- Rust workspace package version is currently `0.6.4`.
- Go module path is `github.com/netdata/systemd-journal-sdk/go` and requires
  `go 1.26`.
- Release tags `v0.7.0` and `go/v0.7.0` do not exist locally or remotely at
  activation time.
- Root and Go submodule tags must point to the same commit.

Inferences:

- The v0.7.0 release should publish the Rust crates and create both root and
  Go submodule tags.
- Node and Python registry publication is still outside this release scope
  because their package versions remain `0.1.0` and no registry publication
  process has been requested for them.

Unknowns:

- Crates.io propagation latency during dependent crate publication. If it
  occurs, retry after the registry index catches up; do not record tokens.

### Acceptance Criteria

- Rust workspace version and publishable internal dependency versions are
  `0.7.0`.
- Rust crates publish dry-run passes in dependency order.
- Rust crates are published to crates.io at `0.7.0`.
- `master` is pushed.
- Annotated tags `v0.7.0` and `go/v0.7.0` exist locally and remotely and point
  to the same commit.
- Release validation and tag hashes are recorded without credential details.
